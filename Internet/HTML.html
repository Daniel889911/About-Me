History And Evolution Of Hypertext Markup Language(HTML)

1.0 Purpose

Hypertext Markup Language(HTML) is used to create and display web pages. It markups the data presented on a webpage by using tags which describes its purpose, appearance, and organisation within the webpage. The browsers request web pages using the HTTP networking protocol. It then receives, reads and interprets the HTML file containing HTML markup language and renders them as multimedia web pages.

HTML is enhanced by incorporating scripting languages like javascriptto affect the way the web page behaves and functions and also a styling language named cascading style sheets(CSS) that enhances the look and design of the web page.

2.0 History

HTML was created by physicist Tim Berners-Lee, a contractor at the European Organisation For Nuclear Research (CERN) in 1980 and he was later joined by CERN data systems engineer Robert Cailiau to further develop this technology. The table below shows the evolution of HTML to the current version HTML 5 :

HTML Version History Table

HTML Version Number Features
HTML 1.0
1993
  • Very limited with only 20 elements, could not change page background, use fonts and tables and forms
HTML 2.0
1995
  • First official set of standards for HTML set by W3C
  • Added ability to change page background, text colour, and use tables
  • W3C(The World Wide Web Consortium) was formed to regulate web standards including HTML 2.0
  • Browsers start to support HTML 2.0
HTML 3.2
1997
  • Included tables, applets, text flow around images, subscripts, and superscripts
  • Supported and approved by Netscape and Microsoft browsers
HTML 4.01
1999
  • Introduction of cascading style sheets(CSS) allowing separation of structure to design elements
  • Provided variations of font, text, colour, and background clearly
  • Allowed CSS to be added in a separate file splitting the code for the structure of the web page from its design and style
HTML 5.0
2014
  • Added physical location to web sites
  • Ability to work even when not connected to the internet
  • Capacity to provide high definition video and graphics
  • Supported by all main browsers : Chrome, Safari, Edge, Firefox, and Opera

3.0 Elements And Tags

An HTML file consists of elements which are distinguished from each other by using tags that informs the browser of the structure, format, and display of the web page. The elements can be structured in a nested hierarchical format. Tags come in pairs with a starting tag and an ending tag.

4.0 Main HTML Elements

Block elements take up a new line and uses the full width of the web page, it has a rectangular structure and does not allow any other element to occupy the same horizontal space it is on.

6.0 The Document Object Model(DOM)

An HTML element is a type of HTML node and can be represented in a tree structure. The HTML file is parsed by the browser and stored in a tree data structure as the Document Object Model(DOM) that is an API for javascript. The HTML elements or nodes contain objects and javascript uses the DOM API to manipulate the objects on the fly without the need to reload the web page.
