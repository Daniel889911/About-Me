Purpose And Evolution Of Javascript

1.0 History

Javascript was first developed in 1995 by Brendan Eich working for Netscape. It was inspired from Java, Scheme, and Self programming languages.

Javascript is a core technology for internet webpages and is used to provide client side web page behaviour. It enables the website to interact and respond to users actions and gives the website a more dynamic and interactive feel to it. All major web browsers have a dedicated javascript engine that can compile javascript. ECMAScript is the official name for Javascript and below is a summary of the version histories condensed from the W3C website.

2.0 Features

Javascript is a high level, imperative, and structured scripting language that uses just in time compilation. Javascript is weakly typed and variables are implicitly cast depending upon the syntax when the type is used. Regular expressions are also supported in the javascript runtime environment.

Javascript uses a prototype style of object oriented programming. Prototype objects are able to make new instances of themselves which are mutable giving them the ability to create new prototype objects with new fields and methods. The prototype objects inherit from their parent objects without the use of classes.

A function in javascript is considered to be first class, that is it is considered as an object and will have properties and methods associated. Functions are also constructors and prefixing a function with the syntax new will create an instance of the named prototype inheriting properties and methods from the constructor.

Javascript processes messages passed between objects and places them in a queue. When the message is processed its function is called creating a stack frame of the functions arguments and local variables. The call stack uses memory as required and when the function has completed the next message in the queue is processed on the call stack. Messages are processed to completion and this is called the event loop and as javascript processes concurrently, the event loop is non blocking, this is achieved through an input/ output system that uses events and callback functions so that other messages can be processed while waiting for an event to happen which initiates when the callback function is ready for message passing.

Promises are a feature of javascript that allow for asynchronous operations. Promises are a built in object that comes with handlers that return back the results of the promise whether it was successful or failed. The returned results from the promise do not have to happen immediately but until the promise's function event has completed, so the promise guarantees to return the result sometime in the future.

3.0 Asynchronous Programming Styles

Unlike other programming environments where each program follows a certain sequence in a predictable way, the web browser environment responds to events that happen that are not predictable. For example the web browser maybe waiting for a mouse click event that may never happen or waiting for a fetch() or XHR request and the server takes a long time to respond or does not respond at all. In order to allow it to continue to process other computations while waiting for an event to happen the web browser uses asynchronous programming concepts. There are two main styles of asynchronous programming styles in javascipt, an older style named callbacks and a newer style called promises.

3.1 Asynchronous Callbacks

The most common form of an async callback function is the addEventListener function :

btn.addEventListener('click', nextItem);

The addEventListener function has two arguments, one of the event which is the ‘click’ and another callback function which is executed once the ‘click’ event has happened. Since the callback function is an argument to the addEventListener function, it is not executed straightaway and it is internally coded by the addEventListener to execute only when the event ‘click’ has happened, in pseudocode this is :

If (event ‘click’) then callback function,

Else continue loop to poll ‘click’ event

3.2 Asynchronous Promises

Promises is the new style of async coding with the common example the fetch() API :

fetch(url).then(function(data) {

  return data.json(); });

The fetch() API takes a url of the server as an argument and returns a promise object. The promise object is the result of whether the request to the server was successful or not. The promise object then has two possible scenarios for the result of the event, which are provided in two methods for the object.

  • The .then method executes code if the promise object request was successful i.e. data was retrieved from the server
  • The .catch method executes code if the promise object request was unsuccessful i.e. data could not be retrieved from the server

Multiple asynchronous events can be chained one after the other as shown below :

fetch(url).then(function(response) {

return response.json() })

.then(function(data) {

console.log(data);

})

5.0 AJAX

Asynchronous Javascript and XML (AJAX) is a technology that allows data to be transferred from the server to the web page in the background without reloading the whole page. It does this by creating an XHR object which requests information from the server, the returned object contains data in JSON format and the required information can be retrieved using the properties method. Javascript can then use the XHR object and its data properties to modify the web page element objects through the HTML DOM manipulation without reloading the whole web page.

Originally the data from the XHR object used to be in XML format, however, nowdays it is more common to have it in JSON format which is a format that is very similar to a javascript data structure.
